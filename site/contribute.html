Contribute

Submit new subagent template versions through pull requests.

The registry accepts versioned template directories only. Existing released versions stay immutable after merge.

Open a PR Back to registry

Submission rules

  1. Create a new directory under `templates/<name>/<version>/`.
  2. Include `subagent.yaml`, prompt assets, schemas, skills, and `README.md`.
  3. Declare `metadata.authors`, `spec.compatibility.hosts`, and any structured dependencies.
  4. Do not modify an existing released version directory. Publish a new version instead.
  5. Let the validation workflow pass before requesting review.

Review flow

`CODEOWNERS` requests a maintainer review automatically. Protected branch settings in GitHub should require at least one approval and a passing validation workflow before merge.

Custom domain

The first release uses the default `github.io` domain. A later phase can add a `CNAME` file and DNS once the site shape is stable.
